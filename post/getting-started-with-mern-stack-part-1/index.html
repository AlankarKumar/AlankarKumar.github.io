Getting Started with MERN Stack Part 1

SetUp a basic MERN stack Application

Posted on August 28, 2017

Hello!

Web development has evolved a lot with recent updates to ECMA script specifications. For those who have heard the term MEAN Stack and are confused as to what does MERN mean. It is just React – R for the Angular – A in the stack, the rest being MongoDB , ExpressJS , NodeJS .

We won’t dive into the pros and cons of React or it’s relative advantages and disadvantages to Angular, because that’s another level of opinionated cacophony of technology loyalists.

This tutorial is the first part of the 3 tutorials in this “Getting Started with MERN Stack” series. I’ll be focusing on creating the minimum working code for setting up the stack and a working application and not be focusing on best practices, styling and validations.

Let’s have the introductions now ….

What is NodeJS?

Node.js is a JavaScript runtime built on Chrome’s V8 JavaScript engine.

Node.js uses an event-driven, non-blocking I/O model that makes it lightweight and efficient. This is what Node’s website says it is.

Basically, NodeJS realized a web server built on javascript, that was an achievement in itself. You can check out Ryan Dahl presentation at Google IO on Node.In this tutorial I’ll be introducing the parts relevant to our application as an when they occur.

What is ExpressJS?

Express is a framework for building web and mobile applications consisting of common backend functionalities and other templating engines.

Because Node.js itself wasn’t intended to build websites, the Express framework is able to layer in built-in structure and functions needed to actually build a site. It’s a pretty lightweight framework that’s great for giving developers extra, built-in web application features and the Express API without overriding the already robust, feature-packed Node.js platform.

What is MongoDB?

Well it is the most popular NoSQL available out there. For your relevance to this tutorial it is the database component of our application.

What is ReactJS?

ReactJS basically is an open-source JavaScript library which is used for building user interfaces specifically for single page applications. It’s used for handling view layer for web and mobile apps.

It uses the concept of Virtual DOM to manage your page and with it’s components gives your webpage a high level of modularity and control. It uses JSX as the language to create components which indeed pretty interesting and powerfull.

What we will build……

We will take a note taking application as our use case with add and delete functionalities. We won’t focus on any CSS or styling and restating we will be writing the minimum amount of code for just getting our application up and working.

What we do in this part of the tutorial…

In this part we’ll go ahead and setup your system with the necessary tools and get the basic skeleton of our application ready with all the necessary files.

What we need?

We need to install Node/NPM.You can download Nodejs from here. NPM is nodejs package manager and it gets installed automatically when you install Node. For more reference on how to install Node you can see the help on Node’s website.

Let’s get started…

We will use facebook’s create-react-app as a starting point as it does a lot of things which would be an overhead for this tutorial. You don’t need to install or configure tools like Webpack or Babel. They are preconfigured and hidden so that you can focus on the code.

   npm install -g create-react-app

The -g is to install it globally so that it is setup in the overall environment and not only in your project folder. It relieves you of the pain to install it again for each and every project that you start. You can find the github repo for create-react-app here.

Navigate to the path wherever you need to create your react app and type the below in your terminal

   create-react-app notes 
   cd notes

where notes is name of our application , you can vouch any other. create-react-app creates your application folder with all necessary node modules, webpack configuration.

Type the npm start in your terminal and you will have a browser window open with facebook’s default react app.

We are inching closer to having our frame ready to design the app.

We’ll do a bit of cleaning up on the default app. Navigate to the src folder and remove all the files except the index.js file. This is the top file which your app ultimately renders alongwith all your components.

In React all the components are mounted to a div with an id. In our case this div lies in the index.html file in the public folder of your react app with id root. Open the index.html and get rid of

   <link rel=”shortcut icon” href=”./src/favicon.ico”>

line in the head. If we draw a react-component level diagram of our application it would be fairly simple.

Notebox is the parent component and renders Note which is the individual notes that user adds. So, root holds the Notebox and Notebox contains Note.

This is the modularity that React offers which makes it very effective to build scalable UI as even the most complex of designs can be broken down into components and divided among teams.

Next Up…

Creation of directory structure for any project is very critical, as the size of projects increase and so do the number of imports, keeping the application scalable yet maintainable becomes very critical. But , 😉 here we are only concerned about getting started and the minimum amount of code required to get a working MERN stack application. So We will keep things simple.

Create the following files in the src folder.

  • index.js
  • Note.js
  • NoteBox.js
  • NoteSchema.js
  • Server.js

For our database we will use MongoDB for which we have two options, either use the local machine installation or use a free hosting such as mlab. I would recommend using a local machine installation as it is generally easier to get going once you have done the setup.

You can download mongodb from here.If you are new to mongodb you might want to take a look at the documentation for basic CRUD opeartions.

Lets setup our other dependencies.Type in the following in the console

   npm i axios express body-parser foreman nodemon marked mongoose –save

I’ll introduce these dependencies whenever put to use.

Also let’s create a Procfile which will start our application as well as the API.We will create this Procfile in our root directory.Type the following in the file

   web: react-scripts start
   api: nodemon server.js

Now head to the package.json file

"scripts": {
    "start": "react-scripts start",
    "start-dev": "nf start -p 3000",
    "build": "react-scripts build",
    "test": "react-scripts test --env=jsdom",
    "eject": "react-scripts eject"
}

and add the start-dev and start as given above.

Now when we run

   npm run start-dev

we will start up both React App as well as the API.But wait You have nothing to render as of now. But you ‘re ready Switch to the next part of this tutorial To get started with coding your application from ground up.

A quick word of advise, before you take up the next post I assume a little familiarity with react’s jsx syntax and the concept of components and props. Facebook React documentation is a wonderful place to learn these concepts.
